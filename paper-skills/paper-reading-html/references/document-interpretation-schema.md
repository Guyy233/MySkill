# Document Interpretation Schema

Use this file for Board 2: Document Interpretation, not for the line-by-line board.

This board works at whole-paper scale.
It should feel like a second reading after the source has already been walked through.

## Two supported modes

### Mode A: Question-driven interpretation

Use this mode when the user already has questions, tasks, or course-facing prompts.

Organize the board around:

- the question
- the paragraph or section that answers it
- the answer itself
- whether the answer is strong, partial, or weak

### Mode B: Pure logic interpretation

Use this mode when the user wants a full-paper logic map without preset questions.

Organize the board around:

- thesis
- problem setup
- method chain
- evidence chain
- weak joints or counterpoints
- final takeaway

## Mandatory slots

No matter which mode you use, the board should answer these slots:

1. **One-sentence thesis**
   State the paper's real claim in one sentence.

2. **What problem is being solved**
   State the target problem and the practical or theoretical pressure behind it.

3. **How the method chain works**
   Explain the pipeline, not just the method name.

4. **Where the real evidence sits**
   Point to the experiments, derivations, ablations, or case studies that actually carry the paper.

5. **Limits or weak joints**
   Name the limits, unsupported leaps, or dependence on specific assumptions.

6. **What the reader should retain**
   Give the end takeaway in plain language.

## Chapter logic map

Add a compact section map that tells the reader what each major section does.
Do not only repeat the section titles.
Explain the role of each section in the paper's argument.

## What to avoid

Do not turn this board into a buzzword list.
Do not repeat paragraph notes.
Do not write empty praise.
Do not hide weak evidence when it matters to interpretation.
