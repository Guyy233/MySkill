# 论文分析

把一篇论文加工成两份可直接使用的中文 PDF：**带读手册**（负责总结、讲解、批判性分析和背景知识）和**逐句中英对照三色标记译文**（负责忠实翻译，保留论文原图，并在对应位置给出双语图注与逐图解读）。两份文件严格分工：除图片的图解外，任何总结、解释或整理者批注都只能进入带读手册，不能混入译文。

## 何时使用

用户要求"带读""精读""翻译并标注""三色批注""整理这篇论文"等，且材料是一篇具体的论文 PDF（或已提供论文文本）时使用。如果只是随口问论文内容（不要求产出文档），不必套用本流程，直接回答即可。

## 总体流程

1. **通读原文，建立真实理解与清单**——直接读取整篇 PDF，并逐页记录章节、正文句子、公式、表格、正文编号图片、子图和图注。文本提取只能辅助定位，不能替代对双栏顺序、公式和图片的视觉核对。翻译前建立按章节排序的句子清单和图片清单，后续逐项销号。
2. **确认交付形式（通常不必再问）**——除非用户明确要求别的格式，默认按以下已验证的组合执行，不必每次都用 AskUserQuestion 确认：
   - 最终产物：**PDF**（用 xelatex + ctex 编译，而不是网页/Word/Markdown——公式是原生 LaTeX 渲染，最不容易出现字母、上下标、希腊字母乱码或错位）。
   - 译文排版：**中英对照、逐句配对**。每个 `origtext` 只放一个完整原句，后面立即放该句的完整中文译文，然后再处理下一句。
   - 只有当用户的措辞明显偏离这个默认（例如只要一份文件、要网页版、要纯中文译文不要对照）时才用 AskUserQuestion 确认。
3. **准备编译环境**——检查 `xelatex`、`kpsewhich ctex.sty`、`kpsewhich xecjk.sty` 是否可用。若不可用，如实告知用户当前环境缺少 TeX 环境，询问是否要退化为 Markdown 交付（并说明这会降低公式/三色高亮的呈现质量），不要静默降级。
4. **搭建工作目录**：在 scratchpad 里新建一个子目录，把 `assets/preamble.tex` 复制进去，作为两份文档共用的导言区。新建 `figures/` 保存从原论文提取的图片。
5. **建立图片清单并提取原图（强制）**——开始撰写两份文档前，完整读取 `references/figure-handling.md`，逐项建立图号、页码、首次引用位置、原始图注、子图、坐标轴、图例和本地文件名的清单。凡正文引用的编号图片都必须从原 PDF 高保真提取到 `figures/`；不得用生成式工具重绘、补字、改色或改变数据。
6. **撰写《带读手册》**，结构对照 `assets/guide_template.tex`：
   - 文章速览（标题/作者/单位/一句话概括/核心创新点）
   - 读前必备：前置知识地图（只列这篇论文真正用到的背景知识，按流水线顺序排列）
   - 术语与符号对照表（longtable）
   - 逐段精读（按原文章节顺序，摘要用"问题→局限→思路→验证"四段式拆解；方法章节公式逐条讲解物理/工程直觉；实验章节讲清楚每个实验验证了什么）
   - 逐图带读（每张正文图片都放在对应正文附近；先教读者怎么看图，再说明坐标轴/图例/子图/关键趋势、作者用它支持什么结论，以及证据的边界）
   - 自测问题（6~10 道，覆盖前置知识/设计动机/公式细节/实验结论/局限展望）
   - 写作层面的启发（这篇论文的结构/论证套路对读者自己写作的借鉴）
   - 全篇要"循循善诱"：多用启发式提问和类比（"你可能会问……""不妨把……想象成……"），不要写成干巴巴的摘要复述。
7. **撰写《逐句中英对照三色标记译文》**——开始前完整读取 `references/translation-fidelity.md`，结构对照 `assets/translation_template.tex`：
   - 严格重复“一个英文原句 → 该句中文译文”的配对单元，不得把整段英文放在一起后再给中文概述。
   - 中文长句可因语法需要拆分，但必须保留该原句的全部命题；不得把多个原句合并为一个摘要。
   - 正文只能出现原文已有内容及其译文。不得添加导读、释义、背景补充、段落总结、译者按、批注小结或任何其他解释。
   - 三色只用于给忠实译出的原句着色，不能添加新句子；不上色不影响翻译完整性。
   - 逐图展开：在正文首次讨论对应图片的段落之后插入原图，依次给出英文原始图注、中文图注和明确标为“图解”的分析框。图片、图号、子图标识和数据必须忠实于原文。
   - 三色分配的默认经验规则：
     - 🟩 **绿色**（研究创新点/核心结论）：摘要与引言中"本文提出/核心思路/贡献列表"；方法章节里被作者明确称为创新设计的关键选择；实验中由结果得出的结论性陈述。
     - 🟨 **黄色**（实验方法/数据模型/研究思路）：方法章节的具体算法、公式、约束设计；实验环境、指标、对比设置等可直接借鉴的技术细节。
     - 🟥 **红色**（争议点/不足/未来展望）：相关工作中对已有方法"指出局限"的句子；对比方法的局限描述；结论中的 future work。

## 逐句翻译的硬边界（强制，不得省略）

`references/translation-fidelity.md` 是译文的验收合同，每次制作译文都必须完整读取并执行。最核心的规则是：

1. **一一对应**：每个原句都必须有紧邻的中文译句，顺序与论文一致；不能只保证“段落大意覆盖”。
2. **不得替代**：summary、改写、批注、术语解释和背景知识都不算翻译，不能替代任何原句。
3. **不得增删**：保留否定、条件、比较、程度、时态、指代、因果、模态、不确定性、数字、单位、引用和公式编号；不得弱化、强化或补写作者没有表达的结论。
4. **内容隔离**：除原图的双语图注与图解外，译文中不出现任何整理者分析。所有非图解释都放进《带读手册》。
5. **先翻译后着色**：先完成并校验全部逐句译文，再按原句性质选择性着色；颜色不能改变、截断或取代译文。

## 图片处理与逐图分析（强制，不得省略）

执行前完整读取 `references/figure-handling.md`。凡正文引用的编号图片，都必须同时出现在《带读手册》和《逐句中英对照三色标记译文》中；补充材料中的图片在用户提供的 PDF 包含补充材料时同样覆盖。装饰性 Logo、出版社页眉和下载水印不属于论文图片。

每张图必须满足以下要求：

1. **位置正确**：放在正文第一次实质讨论该图的段落之后，不能把所有图片集中堆到文末；必要时用 `\FloatBarrier` 防止图片漂移到错误章节。
2. **图像忠实**：直接提取或裁剪原 PDF 中的图片，保留原始标签、颜色、比例和子图布局；只允许无损裁剪空白边缘与等比例缩放。
3. **图注完整**：保留英文原始图注，并紧接着给出忠实的中文图注翻译；不能只写自拟标题代替原图注。
4. **坐标与图例**：若是数据图，逐项说明横轴、纵轴的名称、单位、范围、线性/对数刻度，以及每条曲线、颜色、线型、标记和误差棒代表什么；没有坐标轴的结构图或照片明确写“无坐标轴”，再解释模块、箭头或标注。
5. **主要信息**：说明关键趋势、峰值/拐点/收敛/差距/异常、重要数值和不同方法之间的比较，并回答“这张图主要讲了什么”。多子图必须按 `(a)(b)(c)` 分别解释，再总结它们之间的关系。
6. **证据边界**：区分“图中直接可见”“作者据此声称”和“带读者可合理推断”的内容。分辨率不足或数值不可读时必须说明，不得编造坐标、数值或统计显著性。

《逐句中英对照三色标记译文》中的图解是译文里唯一允许出现的逐图分析，分析框标题只写“图 X 图解”（例如“图 1 图解”），不得显示“整理者分析”“译文唯一的附加分析”等说明性文字，并与英文原始图注、中文图注分开排版。图解不得冒充原文，也不能夹带对非图正文的段落点评。

## 编译与校验（不要跳过）

1. `cd` 到工作目录，`xelatex -interaction=nonstopmode <file>.tex`，连续编译 2~3 遍（目录、交叉引用需要多遍才能稳定）。
2. 每次编译后 `grep -n "^!" <file>.log`——只有以 `!` 开头的才是真正的错误；`Font shape ... undefined` / `Some font shapes were not available` 这类是良性警告，可忽略。
3. **扫描源文件里的可疑字符**，提前发现"编译不报错但显示会丢字"的隐患（例如带圈数字 ①②③ 会被 PingFang SC 静默丢弃，出现"Fig.8 中 – 的连续摆臂动作"这种断裂文本）：
   ```python
   text = open('<file>.tex', encoding='utf-8').read()
   suspicious = {ch for ch in text if ord(ch) > 127
                 and not (0x4E00 <= ord(ch) <= 0x9FFF)
                 and not (0x3000 <= ord(ch) <= 0x303F)
                 and ch not in '（）＋，：；？'}
   print(sorted(suspicious))
   ```
   如果扫出带圈数字、生僻符号等，改写成安全的中文表达（如"第 2–4 张"而不是"②–④"）。
4. **逐句覆盖核对**：用翻译前的句子清单逐项检查。每个原句必须能定位到一个紧邻译句；不得以段落数量接近、关键词出现或 summary 覆盖代替逐句验收。重点复核否定词、条件句、比较级、模态词、数字、单位、引用和公式指代。
5. **正文纯度核对**：除图片、双语图注和明确标识的图解外，搜索并删除译文里的“本段说明”“也就是说”“值得注意的是”“简而言之”“译者注”“小结”等新增解释。若这些文字确实来自原文，则必须保留英文原句并作为普通句子配对翻译。
6. **逐图核对**：把图片清单与两份最终 PDF 对照，确认每张正文图片都出现且只出现一次，图号、子图、英文图注和中文图注匹配原文，坐标与图例在 100% 缩放下可读，没有裁掉标签或让图片漂移到错误章节；每张图后都有实质性图解，而不是只复述图注。
7. **视觉核对**：打开生成的 PDF 实际检查中文、公式、逐句配对、色块、分页和图片可读性，不能只凭编译无报错就交付。

## 已知坑与固定写法（写新模板时直接照做，不要重新踩坑）

- **不要用 `soul` 包做中文内联高亮**：`soul` 的 `\hl` 是按连字规则逐字符处理的，遇到中文会报 `Package soul Error: Reconstruction failed`。三色高亮一律用 `preamble.tex` 里定义好的 `hlg`/`hly`/`hlr` 三个 `tcolorbox` 环境（背景色块 + `breakable`，可在中文段落里正常跨行）。
- **`\newtcolorbox` 的参数一定要写成 `[1][]`（空默认值），不要只写 `[1]`**：`[1]` 是必选参数，调用时必须用花括号 `{标题}`；`[1][]` 才是可选参数，调用时用方括号 `\begin{box}[标题]`。这套模板里所有 `\begin{tip}[...]`、`\begin{term}[...]` 都是方括号调用，配套的 `\newtcolorbox` 定义必须带空默认值，否则方括号里的文字会被当成普通正文、方括号本身也会被打印出来。
- **中文字体只设 `\setCJKmainfont{PingFang SC}`，不要额外设 `\setmainfont`**：让西文默认使用 Latin Modern，字重（粗体/斜体）齐全，避免"Font shape ... undefined"的替换警告；PingFang SC 是 macOS 自带字体，其他系统需换成 Noto Sans CJK SC 等等效字体。
- **`tip`/`term` 这类带标题的色块，标题文字颜色要和标题背景色对比度足够**（例如深蓝底配白字，而不是深蓝底配深蓝字），否则标题会"视觉消失"。
- **`origtext` 一次只放一个原句**，后面立即放该句译文；不要把整段原文和整段译文分别堆放，更不要用概述替代逐句翻译。
- **三色只是原译句的视觉标记，不是新增批注**。译文开头不加三色说明框，结尾不加三色小结，正文不加任何非图解释。
- **不要用生成式图像工具重建论文图片**：论文图是证据，必须来自源 PDF。只允许确定性的提取、裁剪空白边缘、格式转换和等比例缩放；不得修复、补画或重排图中内容。
- **不要只翻译图注而省略图片和图解**：图片、英文图注、中文图注、逐图分析四者缺一不可。

## 输出与命名

最终两份 PDF 放到用户论文所在目录（不要留在 scratchpad），命名为：
- `带读手册-<论文英文标题或简称>.pdf`
- `逐句中英对照三色标记译文-<论文英文标题或简称>.pdf`

## 素材

- `assets/preamble.tex` —— 两份文档共用的导言区（字体、三色标记、原文框、图解框和提示框定义），`\input` 进正文文件即可。
- `assets/guide_template.tex` —— 带读手册骨架，含每节写作要求的注释说明。
- `assets/translation_template.tex` —— 逐句中英对照译文骨架；不包含正文批注说明或文末总结。
- `references/figure-handling.md` —— 强制逐图工作流、放置规则、分析字段和验收清单；每次执行都要完整读取。
- `references/translation-fidelity.md` —— 逐句忠实翻译、禁止增删与句级验收规范；每次执行都要完整读取。
