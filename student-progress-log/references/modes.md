# 模式模板

在用户需要更固定的输出样式时，按下面三种模式选择合适模板。

## 1. 日常记录模式

用于当天工作结束后的简洁记录。

### 模板

### YYYY-MM-DD
- 主要目标：
- 已完成：
- 关键文件：
- 问题 / 卡点：
- 下一步：
- Git 备份：

### 使用要求

- 保持简洁，适合日更。
- 更强调“今天实际做了什么”。
- 如果当天改动明确，写出关键文件或目录。

## 2. 周报模式

用于按周整理阶段进展，适合发给导师、自己做周回顾，或为正式周报准备素材。

### 模板

## YYYY-MM-DD 至 YYYY-MM-DD 周报
- 本周重点：
- 本周完成：
- 关键进展：
- 遇到的问题：
- 当前判断：
- 下周计划：
- 相关文件 / 证据：
- Git 备份：

### 使用要求

- 先回看本周几天的日常记录，再做合并总结。
- 不要把每天内容机械拼接，要提炼“这周真正推进了什么”。
- “当前判断”适合写阶段性认识，例如算法是否有效、实验是否稳定、哪个方向更值得继续。

## 3. logbook 模式

用于写得更正式一些，适合转成实验日志、项目日志或课程要求的 logbook 条目。

### 模板

### YYYY-MM-DD Logbook
- 工作内容：
- 具体操作：
- 结果 / 发现：
- 关键文件或证据：
- 问题与分析：
- 后续计划：

### 使用要求

- 语气比日常模式更正式，但仍保持真实。
- 优先写“做了什么”和“发现了什么”，不要只写感受。
- 如果有实验、代码、图、报告，请尽量落到证据文件。
- 如果信息来自推断，要用保守表达，不要写成已被完全验证的结论。

## 选择建议

- 用户说“记录今天进展”时，用日常记录模式。
- 用户说“帮我写这周总结”时，用周报模式。
- 用户说“帮我整理成 logbook”时，用 logbook 模式。
- 如果用户没明确说明，但语气偏正式、要给导师看、或要用于课程材料，优先考虑 logbook 模式。
